# Contributing to GitHub Repository Visualizer

Thanks for your interest in contributing 🚀

## 🧠 Before You Start

* Check existing issues before creating a new one
* Avoid duplicate work
* Keep changes focused and minimal

## 🛠 Setup

```bash
git clone https://github.com/Anshul2308z/GitHub-Repository-Visualizer
cd GitHub-Repository-Visualizer
npm install
npm run dev
```

## 🌱 Create a Branch

```bash
git checkout -b feature/your-feature-name
```

## ✅ Commit Guidelines

Use clear and meaningful commit messages:

Examples:

* `feat: add contributor graph`
* `fix: resolve API rate limit issue`
* `ui: improve dashboard layout`

## 🚀 Pull Request Process

1. Ensure your code runs locally
2. Keep PRs small and focused
3. Add screenshots if UI changes are involved
4. Link the related issue (if any)

## ❌ What Not To Do

* Don’t submit large, unrelated changes
* Don’t break existing functionality
* Don’t open PRs without testing

## 💡 Good First Contributions

* UI improvements
* Performance optimizations
* Bug fixes
* Documentation improvements

## 🧩 Questions?

Open an issue and ask.
